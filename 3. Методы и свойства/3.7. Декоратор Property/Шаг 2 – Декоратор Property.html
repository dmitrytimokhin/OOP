Декоратор property

Знакомясь с изучением объекта property мы не вольно затрагиваем тему «дескриптора».

Дескриптор представляет собой специальный протокол или интерфейс, который позволяет вам контролировать доступ к атрибутам объекта через методы __get__, __set__ и __delete__.

В python дескрипторы часто используются для переопределения поведения доступа к атрибутам классов. Они позволяют определить пользовательские методы для чтения (__get__), записи (__set__) и удаления (__delete__) атрибутов класса, когда к ним обращаются через экземпляр этого класса.

Исходя из определения дескриптора, давайте дадим новое определение классу property


Класс property является дескриптором, который позволяет нам создавать атрибуты объекта, которые можно получать, устанавливать и удалять. Это означает, что мы можем создать атрибут, который будет вычисляться каждый раз при доступе к нему, а также позволит нам установить различные правила для получения и установки значения.

 

В результате предыдущего урока у нас имеется следующий код реализации класса BankAccount

class BankAccount:
    def __init__(self, name, balance):
        self.name = name
        self.__balance = balance

    def get_balance(self):
        print('Get balance')
        return self.__balance

    def set_balance(self, value):
        print('Set balance')
        if not isinstance(value, (int, float)):
            raise ValueError('Баланс должен быть числом')
        self.__balance = value

    def delete_balance(self):
        print('Delete balance')
        del self.__balance

    balance = property(fget=get_balance,
                       fset=set_balance,
                       fdel=delete_balance)

В последней строке мы создаем свойство balance при помощи функции property, в атрибутах которой мы описываем какой метод должен быть геттером, сеттером и делитером.

Теперь давайте более внимательно взглянем на то, что представляет собой результат вызова property

В консоли мы видим, что в результате вызова property мы получим объект property, у которого есть три метода

  • .getter() позволяет определить геттер
  • .setter() позволяет определить сеттер
  • .deleter() позволяет определить делитер

Результатом вызова каждого метода свойства будет новый экземпляр класса свойства (обратите внимание на разные адреса памяти)

При помощи этих методов мы можем переписать определение свойства balance. Вот так оно было определенно ранее

balance = property(fget=get_balance,
                   fset=set_balance,
                   fdel=delete_balance)

и вот как будет описано через вызов методов

balance = property()
balance = balance.getter(get_balance)
balance = balance.setter(set_balance)
balance = balance.deleter(delete_balance)

Или в функцию property можно сразу передать геттер-функцию и тогда код станет еще короче

balance = property(get_balance)
balance = balance.setter(set_balance)
balance = balance.deleter(delete_balance)

Целиком наш код выглядит сейчас так

class BankAccount:
    def __init__(self, name, balance):
        self.name = name
        self.__balance = balance

    def get_balance(self):
        print('Get balance')
        return self.__balance

    def set_balance(self, value):
        print('Set balance')
        if not isinstance(value, (int, float)):
            raise ValueError('Баланс должен быть числом')
        self.__balance = value

    def delete_balance(self):
        print('Delete balance')

    balance = property(get_balance)
    balance = balance.setter(set_balance)
    balance = balance.deleter(delete_balance)

Теперь создавая новые экземпляры мы можем менять баланс через  свойство с именем , но и к сожалению доступ к методам, которые отвечали за геттеры и сеттеры у нас также остается.

Создаем геттер-свойство через декоратор

Давайте сделаем так, чтобы нам было доступно только свойство по имени balance, а методы

  • get_balance
  • set_balance
  • delete_balance

исчезли из списка вызова экземпляра. Для этого нужно вспомнить тему декораторов

Декораторы — это функции, которые принимают другую функцию в качестве аргумента, добавляют к ней некоторую дополнительную функциональность и возвращают функцию с измененным поведением.

Давайте на примере геттера покажу как использовать декоратор, при этом заменим имя свойства balance на my_balance 

class BankAccount:
    def __init__(self, name, balance):
        self.name = name
        self.__balance = balance

    def my_balance(self): # метод 
        print('Get balance')
        return self.__balance

    my_balance = property(my_balance) # свойство

В последней строке мы декорируем метод my_balance при помощи функции property. После такой манипуляции у экземпляров класса BankAccount доступно только свойство my_balance (обратите внимание на букву p перед именем my_balance, она обозначает свойство). Метод my_balance в списке имен мы не видим

Можем воспользоваться синтаксическим сахаром декораторов и переписать код при помощи оператора @

class BankAccount:
    def __init__(self, name, balance):
        self.name = name
        self.__balance = balance
    
    @property
    def my_balance(self):
        print('Get balance')
        return self.__balance

Код стал короче и лаконичнее

Создаем сеттер-свойство через декоратор

После объявления геттер-свойства оно становится доступно внутри класса по своему имени, в нашем случае my_balance. Объект my_balance представляет собой свойство, а значит у него есть метод setter

И мы будем использовать метод setter для создания сеттера через декоратор

class BankAccount:
    def __init__(self, name, balance):
        self.name = name
        self.__balance = balance

    @property
    def my_balance(self):
        print('Get balance')
        return self.__balance

    @my_balance.setter
    def my_balance(self, value):
        print('Set balance')
        if not isinstance(value, (int, float)):
            raise ValueError('Баланс должен быть числом')
        self.__balance = value


Главное, чтобы у метода сеттера было тоже самое имя, что и у вашего свойства.

 

Создаем делитер-свойство через декоратор

Аналогичным образом через имеющееся свойство вызываем метод deleter в самом декораторе для определения у нашего свойства my_balance функцию для удаления атрибута

class BankAccount:
    def __init__(self, name, balance):
        self.name = name
        self.__balance = balance

    @property
    def my_balance(self):
        print('Get balance')
        return self.__balance

    @my_balance.setter
    def my_balance(self, value):
        print('Set balance')
        if not isinstance(value, (int, float)):
            raise ValueError('Баланс должен быть числом')
        self.__balance = value

    @my_balance.deleter
    def my_balance(self):
        print('Delete balance')
        del self.__balance

Имена всех наших методов имеют одинаковое название my_balance. Это нужно, чтобы функциональность геттера, сеттера и делитера находилась в одном свойстве  - my_balance.
