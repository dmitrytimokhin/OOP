«Минутку, что сейчас только что было? Почему отработал сеттер» - наверно подумали вы.

Давайте еще раз взглянем на этот код

class MagicalBox:
    def __init__(self, contents=None):
        self.contents = contents

    @property
    def contents(self):
        if self._contents == "rabbit":
            return "🐇 A magical rabbit!"
        else:
            return self._contents

    @contents.setter
    def contents(self, new_contents):
        if new_contents == "wishes":
            print("🌟 Your wishes are granted!")
            self._contents = new_contents
        else:
            print("✨ The magic didn't work this time.")
            self._contents = new_contents


box = MagicalBox("rabbit")
print(box.contents)

Все дело в этой строчке внутри __init__

Здесь не создается атрибут contents по той причине, что ниже в коде имеется свойство-сеттер с одноименным названием.

И данный пример показывает, что свойство-сеттеры можно сразу использовать в инициализации, чтобы не допустить попадания ненужных или неправильных значений внутрь ваших атрибутов экземпляра
